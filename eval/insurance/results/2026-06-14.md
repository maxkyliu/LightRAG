## Notes / caveats
- **Refusal is detected from free text.** LightRAG only emits the hardcoded `[no-context]`
  sentinel when retrieval returns *nothing*; normally the LLM produces a natural-language
  refusal ("我沒有足夠的資訊…", "沒有權限…"), per prompt instruction (prompt.py:356). The
  harness matches a curated set of refusal-specific stems. All 5 out-of-corpus questions
  were correctly refused (no fabrication).
- **Citation (72%) is a cross-extractor measurement gap, not a retrieval failure.** Ground-truth
  keywords are mined with `pdftotext`, but LightRAG ingested the PDFs with its own parser, so
  the exact 10-char definition slice sometimes isn't byte-present in the returned chunk even
  when the correct document IS retrieved (retrieval@5 = 95%). Whitespace is already normalized;
  remaining gaps are punctuation/encoding differences (e.g. full-width parens 「（國際）」) and
  chunk-boundary placement. Mining keywords from LightRAG's own ingested chunks would close
  most of this gap.
- Hard cases mirror the reference: ultra-generic terms defined across many docs (我們, 保單),
  and a few terms whose canonical definition lives in a doc not surfaced in the top-5 chunks.

## Reproduce
```
python eval/insurance/build_dataset.py          # -> datasets/insurance_eval.jsonl (48 cases)
python eval/insurance/run_eval.py --mode mix --top-k 5
```
Requires the stack running (`docker compose -f docker-compose.final.yml up -d`).
Env overrides: `LIGHTRAG_BASE_URL`, `LIGHTRAG_QUERY_MODE`, `LIGHTRAG_API_KEY`.
